Diagnostics

Scans Firestore data for known anomalies — duplicate class schedules, duplicate ER room placements, and duplicate attendance logs. Running a scan is read-only; cleanup is a separate, confirmed action.

Duplicate Class Schedules

Groups classSchedules by (studentId, className, teacherLastName, startTime). Any group with more than one document is a leftover from a re-import.
Not run yet.

Duplicate ER Room Placements

Groups erRoomStudents by (studentId, periodKey). A student should appear in at most one ER room per period. Duplicates cause stale/conflicting placements.
Not run yet.

Duplicate Attendance Logs

Groups classAttendanceLogs by (studentId, date, className, startTime). Duplicates typically come from re-propagating a schedule on top of existing logs.

Orphan References

Finds classSchedules, erRoomStudents, and classAttendanceLogs records pointing at student IDs that no longer exist in students.
